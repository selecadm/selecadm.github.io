How to distrust CloudFlare Universal SSL

by Adm Selec

View My GitHub Profile

Certificates for CloudFlare Universal SSL are issued from the following intermediates:

COMODO ECC Domain Validation Secure Server CA 2
COMODO Domain Validation Secure Server CA 2
GlobalSign Organization Validation CA - G2

The first two are believed to be used for CloudFlare Universal SSL exclusively, hence it is safe to distrust them.

COMODO ECC Domain Validation Secure Server CA 2
PEM Base64

COMODO Domain Validation Secure Server CA 2
PEM Base64

How to distrust intermediates on Windows, affecting Internet Explorer and Chromium-based browsers:
1. Download and open the certificate
2. "Install Certificateâ€¦"
3. Next, "Place all certificates in the following store" -> Intermediate Certification Authorities, OK, Next, OK
4. Run certmgr.msc
5. Go to "Intermediate Certification Authorities"
6. Locate the certificate, right-click, Properties
7. "Disable all purposes for this certificate", OK

Is it possible to do the same with Firefox?

Below are the bugs regarding (in)ability to distrust intermediate:
Bug 585352 - Certificate Manager misleads users into thinking that they can distrust CAs and/or intermediates
Bug 642503 - Generic blacklisting mechanism for bogus certs (NSS trust module)
Bug 646205 - Unable to mark intermediate CA certificate as untrusted

How to ensure the intermediates being distrusted?

Using Internet Explorer or a Chromium-based browser, try to open sites below (as of 17 October 2015):
https://certificate.revocationcheck.com/ (for testing COMODO ECC Domain Validation Secure Server CA 2)
https://qwintry.com/ (for testing COMODO Domain Validation Secure Server CA 2)
